Заметки в ходе формирования описания языка smile
====================================================

Что можно сказать о данных. Они либо уже есть, либо их еще нет.
В статически типизированном языке при этом известен тип данных, поступающих на определенный вход некоторого оператора. Однако момент поступления этих данных определяется соответствующим сигналом от источника данных. Этот сигнал формируется, когда данные появились. Исходя из этого приемник данных может формировать решение об их обработке.

Таким образом, динамика функционирования задается сигналами, определяющими выполнение соответствующих вычислений.

Сообщение - это данные, подтвержденные сигналом об их наличии.
Данные характеризуются типом и величиной. Последняя соответствует одному из значений типа.

В принципе с точки зрения языка появление сообщения не является важным. Однако существует сигнальный тип, который может принимать одно значение, что ведет к необходимости фиксации соответствующей константы. Ее обозначение: "!".

Тип «массив» (array) предназначен для описания группы данных одного типа. Во многом он аналогичен использованию многомерных массивов традиционных императивных языков программирования. Массив имеет фиксированные длины по каждому измерению. Описание данного типа на уровне языка программирования задается с использованием следующего синтаксиса:

Массив ::= ИмяТипа «(» Размерность «)»
Размерность ::= Целое { «,» Целое }

Примеры массивов:
::
    A << type int(100)
    B << type bool(30, 40)
    
В текущей версии предполагается ограничиться векторами.
